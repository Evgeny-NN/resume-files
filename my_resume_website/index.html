📁 Проект «WFM»

Автоматизированная система планирования, учета рабочего времени и управления персоналом колл-центров.

  • анализ интеграционного взаимодействия с внешними системами, разработка и согласование ADR;
  • разработка ТЗ на интеграцию с Active Directory;
  • подготовка к запуску мобильного приложения (подготовка проектной документации, организация доступов);
  • проектирование интеграционных решений:
    • с внешней системой управления кадрами StaffDesk через Kafka;
    • с DATA-платформой для передачи статистических данных через Kafka;
    • с S3 хранилищем для обеспечения WFM статистикой из чат-платформы;
  • определение параметров и заказ необходимой инфраструктуры;
  • проведение опытно-промышленных испытаний;
  • передача сервиса на поддержку;
  • дополнительно:
    • проектирование адаптеров для обмена данными с внешними системами;
    • проектирование механизмов повторной отправки данных;
    • разработка требований к системам мониторинга и алертинга.
📁 Проект «Корпоративная база знаний»

Централизованная система управления знаниями для сотрудников контактного центра Ecom.

  • проведение комплексного анализа и сравнительного исследования рынка решений для управления корпоративными знаниями;
  • проектирование технического решения по смене схемы авторизации с прямой интеграции с Active Directory на централизованную корпоративную платформу Keycloak;
  • проектирование резервного сценария аутентификации для пользователей других бизнес-юнитов через Active Directory на случай недоступности основного контура Keycloak.
📁 Проект «Хаб коммуникаций»

Общее корпоративное хранилище обращений со стороны клиентов с разных каналов связи (чаты, телефонии, почта) в группу внутренних контактных центров компании, а также аутсорсинговых, которые работают с этими обращениями.

  • формализация бизнес-требований от сторон;
  • проектирование архитектуры решения (схемы интеграций с системами-источниками, механизмы хранения и обработки данных, требования к API);
  • формализация технических требований (структуры данных, обработка ПДн, схемы межсистемного взаимодействия);
  • проработка FRD, NFR (функциональные возможности, схемы данных, примеры использования, ограничения/допущения; производительность, доступность, безопасность).
📁 Проект «Умный поиск»

Встроенная в интерфейс рабочего места оператора колл-центра поисковая система с применением языковых моделей (LLM) для поиска нужной информации при работе с клиентскими обращениями.

  • разработка решения по архитектуре системы;
  • проектирование оптимизированной базы данных (c семантическим кэшированием для снижения нагрузки на LLM);
  • разработка технической документации (сценарии обработки поисковых запросов);
  • создание API-спецификации сервиса взаимодействия с пользовательским UI и системой генерации ответов.
📁 Проект «Маскирование телефонных номеров и СМС»

Система, которая отвечает за подмену номера (сотрудника/мерчанта/клиента/курьера) на виртуальный номер из пула доступных номеров для повышения уровня безопасности.

  • подготовка проекта маскирования для Самоката и Сберлогистики:
    • анализ бизнес-кейсов, проблем, ограничений;
    • формализация функциональных и нефункциональных требований;
    • расчёт экономики внедрения маскирования для каждого бизнес-юнита холдинга;
    • разработка E2E архитектуры от создания заказа в приложении → до установления телефонной связи между абонентами и его пост-обработки.
  • проведение комплексного анализа и описание схемы межсистемного взаимодействия между разрабатываемым сервисом маскирования, внутренним решением виртуальной АТС и внешними операторами телефонной связи;
  • анализ фишинга в текстах СМС и транскрибированных голосовых звонках на схеме: клиент – сотрудники доставки, мерчанты;
  • подготовка требований к BI-отчётности по совершённым звонкам.
